DeepSeek Prompt 规范

一套完整的结构化提示词指南，涵盖框架、技巧、参数与自检清单，帮助你充分发挥模型能力。

📜 基础原则与官方规范

基础原则

  • 明确性：清晰定义任务目标、输入格式和输出规范，消除语义歧义。
  • 结构性：将复杂任务拆解为清晰、有层次的步骤或模块。
  • 可验证性：设定可量化评估的标准或明确格式，确保输出结果易于验证。

基础指令语法

采用 动词 + 对象 + 约束条件 结构，如 "生成一段关于AI安全的白皮书，包含5个章节"。

System Prompt 规范

  • 必须通过 system 角色发送。
  • 仅定义角色与核心约束，禁止包含具体任务或数据。
  • 建议长度 80–150 字（过长可能失效）。

模型路由选择

  • 简单对话选 deepseek-v4-flash
  • 分析/复杂推理选 deepseek-v4-pro

Prompt 输入规范

  • 使用英文书写提示词更稳定。
  • 避免在提示词末尾添加多余的空格或换行符。
  • 目前 Prompt 长度限制为 32K tokens。

API 调用规范

  • 需在请求中指定模型ID。
  • 合理设置 max_tokens 以防止输出截断。

JSON Output

  • system/user prompt 必须包含 "json" 字样及样例。
  • 需设置 response_format 为 {'type': 'json_object'}。
  • 此功能需搭配 deepseek-chat (V3) 模型使用。

文件上传功能：目前仅在网页版支持，API 调用暂不开放此功能。

🏗️ 结构化提示词框架 (Prompt Frameworks)

选对框架能让 DeepSeek 的输出质量实现质的飞跃。以下是推荐的框架：

框架适用场景核心要素
RTF 简单场景 Role（角色）- Task（任务）- Format（格式）
TRACE 复杂项目 Task（任务）- Request（需求）- Action（行动）- Context（上下文）- Example（示例）
CoT 深度推理 加入"请一步步思考"来引导模型展示推理路径
CO-STAR 内容创作 Context（背景）- Objective（目标）- Style（风格）- Tone（语气）- Audience（受众）- Response（响应格式）
✍️ Prompt 设计核心技巧

1. 角色扮演 (Role Prompting)

用"你是一个[资深Python工程师]..." 开头，通过 #expert: 等标识符触发模型的专家知识。

2. 任务拆解与约束

将任务拆解为 Step 1, Step 2, Step 3...，并使用"禁止"、"必须"等强约束词限定范围。

3. 动态参数注入

在 Prompt 中使用 {{variable}} 语法（如 {{user_name}}）实现模板复用，避免重复编写。

4. Few-Shot 示例

指令后提供 2-3 个高质量示例，并用 --- 等分隔符隔开，帮助模型理解意图。

5. 利用记忆锚点

在长对话开始植入特殊标记（如 [ANCHOR:prompt_v3_core_directive_2026Q2]），防止模型"遗忘"核心指令。

⚙️ 关键参数调优
参数说明推荐值
Temperature 控制随机性 严谨任务（代码生成）0.1；创意任务（写诗）0.9
Top-P 核采样，限制候选词范围 建议不要与 Temperature 同时修改
Max Tokens 限制输出的最大 Token 数 防止过长或截断，根据任务设置
Stop 设定停止词 让模型在合适位置停止输出，实现精确控制
✅ 自检清单
  1. 角色：是否在开头明确定义了角色？
  2. 任务：是否给出了清晰、具体的指令？
  3. 格式：是否指定了明确的输出格式？
  4. 边界：是否用强约束词划定了"禁止"范围？
  5. 参数：针对任务类型，是否设置了合适的参数？
  6. 历史：在多轮对话中，是否引用了关键上文？

遵循这些规范，可以更好地引导模型，使其输出更符合预期，减少调试成本。
