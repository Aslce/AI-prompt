核心结构

黄金四要素 (PTCF)

一个能激发 Gemini 潜能的完美 Prompt，应包含以下四个组件：

  • Persona (角色)： 赋予明确的职业背景或视角。
  • Task (任务)： 动词开头，清晰、无歧义地说明具体动作。
  • Context (上下文)： 提供业务背景、数据来源或目标受众，减少幻觉。
  • Format (格式与限制)： 严格规定输出的视觉结构、字数、语气或边界。
设计心法

5 大设计原则

  • 引导指令代替单纯禁止： 告诉模型“做什么”，比“不要做什么”更高效。
  • 大道至简： 避免层层嵌套的伪代码模板，直白的指令表现最稳定。
  • 使用明确的分隔符： 用 `---` 或 `"""` 将“指令”与“输入数据”隔离。
  • 范例依然是王道： 提供 1-2 个“输入-输出”范例（Few-Shot）。
  • 控制 Token 与变量管理： 使用变量占位符，并明确限制生成长度。
多模态与推理

高级技巧

多模态 Prompt (Multimodal)

在文本中建立“空间或时间锚点”。

  • 图片：“分析左上角红圈标注的元件...”
  • 视频：“基于 01:23 至 01:50 处演示的动作...”

思维链引导 (Chain-of-Thought)

处理复杂推理时，显式引导模型进行分布式思考。

"请一步一步思考，并在给出最终答案前，先列出你的中间推导或分析步骤。"
